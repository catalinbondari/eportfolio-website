Email Classification

Phishing Email Classification with Machine Learning

This project tackles the critical challenge of email phishing detection. It utilizes various machine learning models, including the powerful pre-trained BERT (Bidirectional Encoder Representations from Transformers) model, to distinguish between phishing (spam) and legitimate (non-spam) emails.



Combating Phishing Attempts:

The script equips you with a comprehensive toolkit for evaluating and comparing the performance of different machine learning models on the task of email classification. These models include:

  • Logistic Regression
  • Decision Trees
  • Random Forests
  • Gradient Boosting
  • Naive Bayes
  • Multi-Layer Perceptrons


By analyzing the performance of these models, you can identify the most effective approach for accurately classifying emails and protecting users from phishing scams.

Project Features:

Data Preparation:

The script streamlines data preparation by efficiently loading and preprocessing email data from a CSV file. This includes tasks like label encoding and splitting the dataset for training and evaluation.



Custom Email Dataset Class:

For seamless integration with the BERT model, a custom EmailDataset class is implemented. This class handles email data tokenization, ensuring the data is prepared for the BERT model's input requirements.



Model Comparison and Evaluation:

A core aspect of the project is the ability to train and evaluate various machine learning models on the email classification task. The script facilitates this process for Logistic Regression, Decision Trees, Random Forests, Gradient Boosting, Naive Bayes, and Multi-Layer Perceptrons.



BERT Model in Action:

The project leverages the power of BERT, a state-of-the-art pre-trained model for natural language processing tasks. The script incorporates PyTorch and the Hugging Face Transformers library to train and optimize a BERT model specifically for email classification.



Evaluation Metrics:

To assess the performance of each model during training, the script calculates critical evaluation metrics. These metrics include training and validation loss, accuracy, precision, recall, and F1-score.



Making Predictions:

The script provides a practical predict function. This function allows you to classify new emails as phishing or safe using the trained models. This enables real-time email filtering and protection against phishing attempts.



Saving Trained Models:

For efficiency and reusability, the script saves the trained models' states. This eliminates the need for retraining every time you want to classify new emails. Simply load the saved models and use the predict function for quick and reliable email classification.



Understanding Model Predictions:

The project incorporates the LIME (Local Interpretable Model-Agnostic Explanations) library. This library provides explanations for the model's predictions on specific emails. By understanding the factors influencing the model's decisions, you can gain valuable insights into how the model identifies phishing emails.

    Time Series Features

    Time Series Features Project: Activity Recognition with Accelerometer Data

    This project tackles the exciting challenge of recognizing activities using time-series data collected from accelerometers. It accomplishes this by extracting features from segmented data and employing machine learning models for classification.



    Empowering Activity Recognition:

    If you're looking to leverage the power of accelerometer data to identify physical activities, this project provides a solid foundation. It equips you with the tools to translate raw sensor data into meaningful insights about human movement.



    Getting Started:

    To get up and running, ensure you have the following Python libraries installed:

    • pandas (https://pandas.pydata.org/)
    • numpy (https://numpy.org/)
    • matplotlib (https://matplotlib.org/)
    • scikit-learn (https://scikit-learn.org/)

    With these libraries in place, you're ready to explore the script and unlock the potential of accelerometer data for activity recognition.



    Project Core Functionalities:

    Data Loading:

    The script assumes your data is organized within a folder named "data" inside the project directory. It iterates through subfolders within "data," reading all CSV files. Each CSV file is expected to contain two crucial pieces of information:

    • Accelerometer data (X, Y, and Z axes) capturing movements.
    • A designated column named "type" that specifies the activity type for each data point.


    Data Preprocessing:

    The script incorporates two key preprocessing steps to prepare the data for machine learning models:

    • Concatenation: All data points from the CSV files are merged into a single Pandas DataFrame for efficient processing.
    • Standardization: A technique known as standardization is applied using a StandardScaler object. This ensures the accelerometer data (X, Y, and Z axes) is on a common scale, leading to better model performance.


    Feature Engineering:

    This stage involves transforming the raw accelerometer data into features that are more suitable for machine learning models to understand activity patterns.

    • Segmentation: The script divides the data into fixed-length segments, creating windows of accelerometer data for analysis.
    • Feature Extraction: To capture the characteristics of each segment, the script calculates a comprehensive set of statistical features. These features include mean, standard deviation, median, minimum, maximum, range, and interquartile range (IQR) for each accelerometer axis (X, Y, and Z).
    • Feature Storage: The extracted features are meticulously stored in a dictionary and subsequently appended to a list named "segments." Additionally, the activity type corresponding to the first element within each segment is accumulated in a separate list named "labels."
    • Feature DataFrame: Finally, a DataFrame named "features_df" is constructed from the "segments" list. The "labels" list is incorporated as a new column named "type" within this DataFrame, providing a clear association between features and activity types.


    Machine Learning Models:

    The script evaluates the performance of two machine learning models that are well-suited for activity classification tasks:

    • Random Forest Classifier: This is a popular ensemble method known for its effectiveness and robustness in various classification problems.
    • Linear Support Vector Classifier (SVC): This is a powerful linear classifier that excels at separating data points into distinct classes, making it a strong candidate for activity recognition.

    Revolut Attrition

    Revolut Customer Churn Analysis Project

    This project tackles customer churn prediction for Revolut, a popular financial technology company. By analyzing their customer churn dataset, the project aims to develop models that can identify users at risk of leaving the platform.



    Understanding Customer Behavior:

    The analysis starts by exploring the provided customer churn dataset. This data includes demographics, account details, and most importantly, a churn status indicator for each customer.



    Data Visualization:

    To gain insights into customer behavior, the project utilizes various visualizations to explore the distribution of data points. These visualizations include distributions of factors like:

    • Customer age
    • Gender
    • Number of dependents
    • Education level
    • Marital status
    • Income category
    • Card category
    • Months as a Revolut customer
    • Total number of accounts held
    • Months of inactivity in the past year
    • Credit limit
    • Total transaction amount
    • The proportion of churning vs. non-churning customers


    Data Preparation:

    Before using the data for model building, several preprocessing steps are applied:

    • Categorical features, like education level, are converted using one-hot encoding.
    • SMOTE (Synthetic Minority Oversampling Technique) is used to address the imbalanced class distribution, where churners might be a smaller group compared to non-churning customers.
    • Principal Component Analysis (PCA) helps reduce the dimensionality of the data, especially after one-hot encoding, which can create many new features.


    Building Predictive Models:

    The project evaluates two machine learning classification models - Random Forest and Support Vector Machine (SVM) - to predict customer churn. These models are assessed using a technique called 5-fold cross-validation, ensuring reliable evaluation.



    Model Performance:

    The F1 score is used as the primary metric to compare the performance of the two models in predicting customer churn.
